Research-Oriented University Information System

Console-based Java application implementing academic management, research tracking, user authentication, and data persistence using four industry-standard design patterns.

01 — Architecture
Design Patterns
Four industry-standard patterns implemented throughout the system
◈
Singleton
One DataStore instance only. Private constructor. getInstance() creates lazily. Full serialization — save to disk, load back.
DataStoregetInstance()save()load()
◇
Factory
UserFactory.createUser(type,...) returns the correct subclass by type string. Throws IllegalArgumentException for unknown types.
UserFactorycreateUser()StudentTeacher
◉
Observer
News is Subject — holds List<Observer>. NewsSubscriber implements Observer.update(). publishNews() fires notifyObservers().
NewsObserverNewsSubscribernotifyObservers()
◫
Strategy
printPapers(Comparator c) accepts any comparator at runtime. 5 interchangeable strategies: citations, date, pages, GPA, name.
PaperByCitationsPaperByDatePaperByPagesStudentGpa

08 — Error Handling
Custom Exceptions
Domain-specific checked exceptions for business rule violations
InvalidSupervisorException
extends Exception
Thrown when a research supervisor's h-index is less than 3. Only 4th-year students require a supervisor.
⚡ Student.assignResearchSupervisor() · ResearchUtils.assignSupervisor()
NonResearcherJoinException
extends Exception
Thrown when an object that doesn't implement Researcher tries to join a ResearchProject. instanceof check at runtime.
⚡ ResearchProject.joinProject(Object person)
